# 快速扫描模式引擎

> **解决什么问题**：用户知道大概哪里有问题，但不确定是不是最大的问题。用5-8个关键问题快速定位最可能的2-3个问题方向，让用户选择深入哪个。
> **什么时候用**：用户选"快速扫描"模式时启动。
> **联动文件**：快速扫描完成 → 路由到 `modules/` 对应模块 → 进入问诊模式的 Phase 4

---

## 快速扫描流程状态机

```
SCAN-INIT → PROBING → FOCUSING → ROUTING
```

| 当前状态 | 触发条件 | 下一状态 | 动作 |
|----------|----------|----------|------|
| **SCAN-INIT** | 用户选择快速扫描 | PROBING | 说明扫描流程，开始第1个关键问题 |
| **PROBING** | 用户回答了问题，但需要更多信息 | PROBING | 进入下一个关键问题（每次只问1个） |
| **PROBING** | 用户回答中出现明确方向信号 | FOCUSING | 汇总回答，圈定2-3个可能方向 |
| **FOCUSING** | 用户选定一个方向 | ROUTING | 路由到对应模块，进入诊断 |
| **FOCUSING** | 用户说"都不对" | PROBING | 回到第1个问题，换一轮 |

---

## Phase S-1：快速定位（5-7个关键问题）

### 关键问题列表

**每次只问1个问题，等用户回答后再问下一个。**

1. "你最头疼的一件事是什么？用一句话说。"
2. "这件事持续多久了？还是一直如此？"
3. "你已经试过什么方法？效果怎样？"
4. "如果不解决这件事，最坏会怎样？"
5. "你觉得根因是什么？（别担心说不准，随便说）"
6. "如果只能解决一件事，你选哪个？"
7. "你现在同时在做几件事？"

### 状态流转规则

- 每回答1个问题 → 判断是否进入下一问题或提前聚焦
- 出现以下信号时，可提前进入 Phase S-2：
  - 用户明确说出问题领域（"我觉得是商业模式问题"）
  - 用户给出具体数据或明确判断
  - 追问已超过5轮

---

## Phase S-2：方向聚焦

根据 Phase S-1 的回答，圈定最可能的2-3个问题方向：

```
根据你的描述，最可能的问题方向有 2-3 个：

1. **[方向A-商业模式]**：你的描述涉及收入/定价/盈利/成本，可能是商业模式问题
2. **[方向B-增长]**：你的描述涉及用户/获客/留存/转化，可能是增长问题
3. **[方向C-组织]**：你的描述涉及团队/执行/目标/协作，可能是组织问题
4. **[方向D-品牌营销]**：你的描述涉及品牌/认知/定位/竞争，可能是品牌营销问题

你觉得哪个最接近？选一个我们深入。
```

### 方向判断参考

| 关键词 | 指向方向 |
|--------|----------|
| 赚钱、盈利、定价、成本、现金流 | 商业模式 |
| 用户、获客、留存、增长、转化 | 增长 |
| 团队、执行力、目标、OKR、管理 | 组织 |
| 品牌、定位、认知、竞争、营销 | 品牌营销 |
| 产品、功能、体验、迭代、PMF | 产品 |
| 战略、方向、竞争、差异化 | 战略 |
| 定价、盈利、现金流、毛利率 | 财务 |

## Phase S-3：转入问诊

用户选定方向后，路由到对应 `modules/` 模块，进入问诊模式的 Phase 4（模块诊断）。

### 路由表

> **完整路由表详见 `engine/identity-router.md`（唯一权威路由源）。本文件不重复定义。**

### 特殊情况处理

- 如果用户说"都不对"或"说不清" → 回到 Phase S-1，重新问一轮问题
- 如果用户选的方向需要多个模块 → 先进入主模块，诊断后再决定是否需要补充模块

## 快速扫描示例

### 示例：用户说"我业绩不好"

```
AI：快速扫描模式下，我会用几个关键问题帮你定位最可能的问题方向。

Q1: "你最头疼的一件事是什么？用一句话说。"

用户：业绩不好，赚钱难

AI：Q2: "这件事持续多久了？还是一直如此？"

用户：今年开始下滑，之前还可以

AI：Q3: "你已经试过什么方法？效果怎样？"

用户：降价促销了，但利润更差了

AI：Q4: "如果不解决这件事，最坏会怎样？"

用户：可能现金流会出问题

AI：Q5: "你觉得根因是什么？"

用户：我觉得是产品卖不上价，不是用户的问题

---

AI：根据你的描述，最可能的问题方向有：

1. **商业模式** — 你提到"卖不上价"、"降价后利润更差"，这更像是定价/盈利模式问题
2. **品牌营销** — 产品价值感知不够，导致只能靠降价竞争
3. **增长** — 获客效率可能有问题

你觉得哪个最接近？选一个我们深入。
```

## 快速扫描节奏控制

1. **每个问题后等待用户回答**，不要连续追问
2. **最多问7个问题**，超过后强制进入方向聚焦
3. **方向聚焦后等待用户选择**，不要替用户做决定
4. **用户犹豫时**，用"如果只能选一个，你会选哪个？"帮助决策
