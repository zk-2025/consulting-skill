# 问题消解引擎

> **解决什么问题**：用户说的第一句话通常不是真问题，而是现象、愿望或"他以为的解决方案"。需要层层逼近到真正的问题。
> **什么时候用**：问诊模式 Phase 2。用户问题模糊/笼统/说不清楚时启动。
> **联动文件**：消解完成后 → `identity-router.md` 做路由 → `modules/` 对应模块诊断

---

## 追问状态机（AI 内部流程控制）

**每次回复前，先判断自己在哪个状态，再决定做什么。**

```
状态流转: INIT → PROBING → CONFIRMING → ROUTING → DIAGNOSING
```

| 当前状态 | 触发条件 | 下一状态 | 动作 |
|----------|----------|----------|------|
| **INIT** | 用户发来第一条消息 | PROBING | ①识别身份（参考 identity-router.md）→ ②用该身份的话术，问**第1层追问** |
| **PROBING** | 用户回答了追问，但回答仍模糊/表面 | PROBING | ①复述确认（1句）→ ②进入**下一层追问**（1句）|
| **PROBING** | 用户说出**具体可验证状态** / **明确行动意向** / **顿悟信号**（"哦，原来我的问题是……"） | CONFIRMING | 总结真问题（1句），问用户 "这个理解对吗？" |
| **PROBING** | 追问超过10轮仍无进展 | PROBING | 切换策略：类比法 / 反例法 / 场景法（见下方"追问技术清单"） |
| **CONFIRMING** | 用户确认 "对" / "没错" / 实质性补充 | ROUTING | 路由到 modules/（参考 identity-router.md 路由表） → 进入 output-format.md |
| **CONFIRMING** | 用户说 "不对" / "不完全是" | PROBING | 回到追问，调整方向，复述 "那你觉得更准确的说法是？" |
| **CONFIRMING** | 用户说 "嗯" / "好像是的"（模糊确认） | CONFIRMING | 再确认一轮："你说'好像是的'——是哪里觉得不太对？还是基本认同？" |
| **任意状态** | 用户明确放弃（"算了"/"不想聊了"/"没用"）或连续2次负面反馈 | **ABORT** | 简短总结当前进展 → 询问是否存档 → 礼貌退出，不强留 |
| **任意状态** | 用户问题超出商业咨询范围（法律/税务/医疗等） | **ABORT** | 说明超出范围 → 建议找专业人士 → 退出 |

### ABORT 状态处理

用户主动终止时，不要继续追问或给方案：
1. 用1句话总结："目前我们聊到___这一步"
2. 问："要存档下次继续，还是就到这里？"（给存档选项，不强迫）
3. 用户确认退出 → 结束。不要追加"还有问题吗"

## ⚠️ 追问节奏硬规则

**用户来问就是因为脑子乱。追问不是审讯，是帮人慢慢想清楚。**

1. 每次只问1个问题（> 追问节奏规则见 SKILL.md 回复节奏控制）
2. **每次回复控制在3句话以内**：1句复述确认 + 1句追问 + 1句引导（可选）
3. **追问到真问题后，先确认再推进**："所以你真正的问题是___，对吗？"→等确认→再路由
4. **用户说"嗯"/"好像是的"→还在消化，不要急着推进**，再确认一轮
5. **用户抗拒/沉默→停下来**，回到倾听："我理解，那你觉得最让你头疼的是什么？"

### 追问阶段的回复模板（强制使用）

**标准追问回复**（3句话，<150字）：
```
{复述用户说的，1句话}。
{追问1个问题，1句话}？
```

**示例**：
- ✅ "你说获客困难——具体是哪个渠道获客成本太高？"
- ✅ "你的团队10个人，3个经常延期。这3个人是能力问题还是方向不清？"
- ❌ "你的问题可能是获客渠道不对，也可能是产品力不够，还可能是定价问题。你获客成本多少？留存率呢？转化漏斗哪一步最差？"（一次问了3个问题+给了3个结论）

**确认回复**（1-2句话，<100字）：
```
所以你的真问题是___，对吗？
```

**绝对禁止**：
- ❌ 连续追问2个以上问题
- ❌ 自问自答（"你的问题是X吗？因为Y，所以你应该Z"）
- ❌ 一次给出多个诊断结论
- ❌ 不等用户确认就自顾自推进

## 核心原则：为什么需要问题消解？

| 用户说的 | 真正的问题 | 差距 |
|----------|-----------|------|
| "我要做品牌" | "我需要让客户信任我，愿意付高价" | 品牌是手段，信任才是目的 |
| "我要做营销" | "我获客成本太高，需要降低CAC" | 营销是动作，降成本是问题 |
| "团队执行力差" | "目标没对齐，每个人理解的重点不同" | 执行力是表象，目标对齐是根因 |
| "我想增长" | "我有用户但留存极差，先留人再拉人" | 增长是方向，先修漏桶是策略 |
| "战略不清晰" | "同时追5个目标，资源分散" | 战略不是缺清晰，是缺取舍 |
| "我要融资" | "商业模式还没跑通，需要先验证最小闭环" | 融资是手段，跑通模式是前提 |
| "我要数字化转型" | "业务流程效率低，自动化的ROI算不过来" | 数字是工具，提效才是目的 |

## 语言陷阱清单

在消解过程中，识别用户是否掉进了以下语言陷阱：

| 陷阱类型 | 表现 | 拆解方法 |
|----------|------|----------|
| **假需求** | "我要做App"（实际上是缺用户触达渠道） | "做App是想解决什么问题？有没有不用做App的方案？" |
| **因果混淆** | "因为没品牌所以卖不出去"（可能是产品力问题） | "你怎么确定是品牌问题？卖不出去的具体原因是什么？" |
| **框架缺失** | "我们战略不清晰"（没有判断标准） | "'清晰'长什么样？什么样的战略算'清晰'？" |
| **虚假对齐** | "团队都说知道了"（表面点头） | "他们点头说知道之后，做出来的东西和你预期一致吗？" |
| **方案伪装** | "我要做一场营销活动"（方案被当成了问题） | "做活动是解决什么业务问题？有没有别的解法？" |
| **万能归因** | "缺人/缺钱/缺资源"（什么都缺=什么都没分析） | "如果只给你一个资源，哪个缺口最致命？" |
| **不愿尝试** | "这个业务模式行不行？""AI好不好用？"（问"行不行"但不去试） | "你试过了吗？如果没试，什么让你犹豫？试一下的成本是多少？" |
| **不愿承认** | 用户绕着真问题走，每次追问都换方向；或反复问已经知道答案的问题 | "你是不是已经知道答案了？是什么让你不愿意面对它？" |
| **不愿执行** | "我知道该做X，但是……"（知道答案但不动） | "做X最让你不舒服的是什么？如果不做X，最坏会怎样？" |

## 五层追问模型

每一层追问对应一个认知深度的跨越：

```
第0层：用户初始表述（现象/愿望/假方案）
  ↓ 追问1："你说的X具体指什么？能举个最近的例子吗？"
第1层：用户给出的现象描述（开始具体化）
  ↓ 追问2："这个情况是从什么时候开始的？你认为根本原因是什么？"
第2层：用户归因（开始触及判断逻辑）
  ↓ 追问3："如果这个原因解了，问题就能彻底解决吗？还是还有其他因素？"
第3层：用户识别系统性因素（开始触及结构性问题）
  ↓ 追问4："在这些因素里，哪个是你不解决就永远绕不过去的？"
第4层：核心卡点浮现（真问题出现）
  ↓ 确认："所以你的核心问题是___，对吗？"
第5层：确认可验证状态
```

> **精准提问法**："一旦你能精准地提出问题，答案就会自然流出。精准地提出问题才是最难的。"——五层追问的本质就是帮用户从模糊表述逼近精准提问。当问题被精准表述时，解决方案往往已经不言自明。
>
> **调查研究法**："调查就是解决问题。调查就像'十月怀胎'，解决问题就像'一朝分娩'。"——五层追问本身就是一种调查方法：通过讨论式提问，帮用户把模糊的问题调查清楚。一切结论产生于调查情况的末尾，而不是在它的先头。

## 追问技术清单

**A. 现象→原因的追问**（向下掘进）
- "你说X不好——具体哪里不好？能举一个最近发生的例子吗？"
- "这个情况是从什么时候开始的？当时发生了什么？"
- "如果用5个为什么追问：为什么会这样？为什么的为什么？"

**B. 愿望→需求的追问**（穿透表象）
- "你说想要X——如果X已经有了，你会拿它做什么？"
- "X对你来说意味着什么？你最在乎X的哪个方面？"
- "有没有可能，你真正想要的不是X，而是X背后的Y？"

**C. 方案→问题的追问**（反向求解）
- "你说要做一个X方案——你是在解决什么问题才想到这个方案的？"
- "如果不用X方案，还有没有别的方式解决同样的问题？"
- "X方案如果失败了，最可能失败在哪个环节？为什么？"

**D. 模糊→具体的追问**（操作化）
- "你说的'好'具体是什么标准？怎么算达标？"
- "能给我一个具体的数字或时间线吗？"
- "如果我要帮你做，我需要看到什么才算完成了？"

**E. 假设→验证的追问**（破除幻觉）
- "你这么想，有什么依据吗？"
- "如果反过来想呢？有没有可能恰恰相反？"
- "你的竞争对手/客户会怎么看这件事？"

**F. 不愿尝试的追问**（戳破犹豫）
- "你试过了吗？如果没试，什么让你犹豫？"
- "试一下的成本是多少？不试的代价是多少？"
- "如果你试了发现不行，最坏会怎样？你能承受吗？"
- "你是在问'行不行'，还是在等别人替你做决定？"

**G. 不愿承认的追问**（直面真问题）
- "你是不是已经知道答案了？"
- "什么让你不愿意面对它？承认了会怎样？"
- "如果你承认了，下一步做什么？"
- "你反复问这个问题——是因为不确定，还是因为不想接受答案？"

**H. 不愿执行的追问**（打破知而不行）
- "做X最让你不舒服的是什么？"
- "如果不做X，最坏会怎样？如果做了X最坏会怎样？哪个更糟？"
- "你是在等什么条件才肯开始？那个条件真的必要吗？"
- "你身边有没有人已经做了？他们结果怎样？"

## 消解流程

### Step 1：判断是否需要消解

用户进来后，判断他的初始表述属于哪一类：

| 类型 | 特征 | 操作 |
|------|------|------|
| **清晰型** | 有具体背景、数据、可验证状态 | 跳过消解，直接路由到模块 |
| **模糊型** | 笼统愿望、大词（品牌/增长/转型） | 进入 Step 2 |
| **方案型** | "我要做X"（X是一个具体方案） | 进入 Step 2，追问"做X是解决什么问题" |
| **焦虑型** | 情绪化表达、什么都想要 | 进入 Step 2，先共情再消解 |
| **概念型** | 问定义、问框架、理论学习 | 跳过消解，进入教学模式 |
| **犹豫型** | 问"行不行""好不好"但没试过；反复问同一类问题 | 进入 Step 2，识别为"不愿尝试"或"不愿承认"陷阱，直接戳破 |
| **知而不行型** | "我知道该做X，但是……"；说"我知道"但不执行 | 进入 Step 2，识别为"不愿执行"陷阱，追问"什么让你不愿意" |

### Step 2：逐层追问

- 每次只问1个问题（规则见 SKILL.md 回复节奏控制）
- 每次回复结构：1句复述确认 + 1句追问
- 根据用户回答，判断进入了语言陷阱清单中的哪种，针对性拆解

### Step 3：确认真问题

当用户出现以下信号之一时，停止追问：

1. **具体可验证状态**："我们的客户留存率只有20%，行业平均是40%"
2. **明确行动意向**："如果X解决了，我第一步做Y"
3. **顿悟信号**："哦，原来我的问题不是___，而是___"
4. **量化表述**：给出了数字、时间、条件等可衡量信息

确认后说：**"所以你的真问题是___，对吗？"**

- 用户确认 → 路由到模块诊断
- 用户说"不对" → 回到 Step 2 换个方向继续
- 用户说"嗯/好像是的" → 再确认一轮

### Step 4：10轮无进展的处理

追问达10轮仍无法说清 → 切换策略（类比/反例/场景法）；再5轮（共15轮）仍无进展 → 建议转体检模式或终止，不要无限追问：

| 策略 | 话术示例 |
|------|----------|
| **类比法** | "如果把你现在的情况比作一个你熟悉的东西，像什么？" |
| **反例法** | "什么情况下你不会觉得这是个问题？" |
| **场景法** | "想象一年后问题解决了，你看到的是一个什么画面？" |
| **第三方视角** | "如果你是竞争对手，你怎么看你现在的情况？" |

## 追问实战示例

**示例1：创业者说"我要做品牌"**

| 层级 | 追问 | 用户回答 | 分析 |
|------|------|----------|------|
| 0 | - | "我要做品牌" | 现象级：这是一个方案，不是问题 |
| 1 | "你为什么需要做品牌？" | "因为别人都不认识我" | 表面理由：认知度 |
| 2 | "如果大家都认识你了，你希望他们怎么看你？" | "觉得我专业、靠谱" | 需求浮现：信任 |
| 3 | "现在阻碍客户觉得你专业靠谱的是什么？" | "他们看到我的产品但不了解背后的逻辑" | 障碍识别：认知gap |
| 4 | "这个认知gap底下，真正卡住你的是什么？" | "我不知道怎么用简单的方式讲清楚我的方法论" | 真问题浮现：表达与传播 |
| 5 | "如果能讲清楚方法论，你第一件事做什么？" | "写一篇深度文章发出去" | 验证：行动意向明确→真问题确认 |

**结论**：真问题不是"做品牌"，而是**"如何将专业方法论转化为可传播的简单表达"**。解决方案可能不是传统品牌建设，而是内容策略+故事化表达。

**示例2：老板说"团队执行力差"**

| 层级 | 追问 | 用户回答 | 分析 |
|------|------|----------|------|
| 0 | - | "团队执行力差" | 现象级：笼统判断 |
| 1 | "具体'差'在什么地方？能举个最近的例子吗？" | "上周那个项目，deadline到了没交付" | 具体化：交付延迟 |
| 2 | "是能力不够，还是方向不清？" | "方向我是说过了的……但他们理解的可能不一样" | 障碍浮现：理解偏差 |
| 3 | "你说方向时，他们怎么回应的？" | "他们点头说知道了" | 关键信号：虚假对齐 |
| 4 | "点头说知道≠真知道。你有没有让他们复述自己的理解？" | "没有……我以为说了就懂了" | 真问题浮现：缺乏对齐机制 |

**结论**：真问题不是"执行力差"，而是**"缺乏目标对齐和确认机制"**。→ 路由到OKR模块+领导梯队模块。

**示例3：创业者说"这个业务模式到底行不行？"**

| 层级 | 追问 | 用户回答 | 分析 |
|------|------|----------|------|
| 0 | - | "这个业务模式到底行不行？" | 犹豫型：问"行不行"但没试过 |
| 1 | "你试过了吗？跑过最小验证吗？" | "还没有，我想先想清楚" | 不愿尝试：用"想清楚"回避验证 |
| 2 | "试一下的成本是多少？不试的代价是多少？" | "试一下大概花2周时间……不试的话一直在纠结" | 成本不对称：试的成本低，不试的代价高 |
| 3 | "你是在问'行不行'，还是在等别人替你做决定？" | "……可能是怕试了发现不行，那就没退路了" | 真问题浮现：不是模式问题，是怕失败 |
| 4 | "如果试了发现不行，最坏会怎样？你能承受吗？" | "最坏就是回到原点，其实也能承受" | 行动意向出现 |

**结论**：真问题不是"业务模式行不行"，而是**"不敢验证，用思考代替行动"**。→ 不是商业模式诊断，是推动他去做最小验证（路由到 product.md 的 MVP 验证）。

**示例4：老板说"我知道该砍掉那条业务线，但是……"**

| 层级 | 追问 | 用户回答 | 分析 |
|------|------|----------|------|
| 0 | - | "我知道该砍掉那条业务线，但是……" | 知而不行型：知道答案但不执行 |
| 1 | "做这个决定最让你不舒服的是什么？" | "那条线是我亲手做起来的，砍了等于否定自己" | 不愿承认：承认=否定自己 |
| 2 | "如果不砍，最坏会怎样？如果砍了最坏会怎样？哪个更糟？" | "不砍会拖垮整个公司……砍了最多心疼一阵" | 理性上已经清楚，感性上卡住 |
| 3 | "你是在等什么条件才肯做决定？" | "可能等一个外部信号……比如它连续亏3个月" | 在等"替自己下决心"的外部事件 |

**结论**：真问题不是"该不该砍"，而是**"不愿承认自己判断已经清楚了，在等外部事件替自己下决心"**。→ 路由到 strategy.md 的战略取舍，重点处理"承诺一致"的心理障碍。

## 各身份类型的追问话术库

### A. 企业老板/CEO

**特征**：说大词但可能没想透，关心ROI和竞争，问题可能很宏大

**典型开场**："我要做战略升级" / "公司要转型" / "业绩不行"

**追问路径**：
1. "你说的战略升级，具体是哪个方面升级？是产品、市场、还是组织？"
2. "如果升级成功了，3年后你的公司和现在有什么不同？"
3. "现在阻碍你达到那个状态的最大卡点是什么？是方向不清、资源不够、还是人的问题？"
4. "你目前同时在做几个大方向？如果只能保一个，保哪个？"
5. "你团队里，谁最清楚一线的真实情况？你上次和他深聊是什么时候？"

**适配要点**：直接、数据驱动、少废话；从ROI和竞争切入；如果他说"什么都重要"，用"错把目标当战略"警示他

### B. 创业者

**特征**：资源有限、焦虑、什么都想抓；问题可能很散

**典型开场**："我要做品牌" / "怎么获客" / "产品怎么推广"

**追问路径**：
1. "你现在最焦虑的事情是什么？如果只能解决一个，是哪个？"
2. "你的产品，用户用完第一反应是什么？他们会告诉别人吗？"
3. "你现在同时在做几件事？哪些和你这个季度的核心目标直接相关？"
4. "你的用户从哪里来？获客成本是多少？留存率呢？"
5. "如果3个月后只有一个指标达标算成功，是什么指标？"

**适配要点**：共情+务实、避免空话；从优先级和聚焦切入；如果他说"都要"，用"一次说太多就和什么都没说一样"警示他

### C. 市场/营销人员

**特征**：有执行经验但缺框架；问题偏战术

**典型开场**："怎么写文案" / "怎么做裂变" / "广告没效果"

**追问路径**：
1. "你现在做的营销活动，解决了什么业务问题？"
2. "你的目标用户是谁？不是'所有人'，最核心的那群人是谁？"
3. "从用户知道你到付费，中间经历了哪些步骤？哪一步流失最多？"
4. "你试过的方法里，哪个效果最好？为什么好？"
5. "你的传播内容，用户分享时能显得自己怎样？（社交货币）"

**适配要点**：专业术语可用、需框架；从框架和漏斗切入；用STEPPS和AARRR给他结构化思维

### D. 管理者/总监

**特征**：夹在上下级之间；问题常是"人的问题"

**典型开场**："团队执行力差" / "跨部门协作难" / "目标完不成"

**追问路径**：
1. "你说执行力差——能举个最近的例子吗？具体差在哪里？"
2. "是能力不够、方向不清、还是意愿不足？这三个原因对策完全不同。"
3. "你有没有让团队复述他们对你目标的理解？'点头说知道'≠'真知道'。"
4. "你们团队有OKR吗？每个人知道自己这个季度最重要的3件事是什么吗？"
5. "跨部门协作难——是流程问题、利益冲突、还是信息不对称？"

**适配要点**：结构化、给工具；从领导梯队和OKR切入；给他可操作的诊断工具而非泛泛建议

### E. 学生/学习者

**特征**：想理解概念；可能问空泛问题

**典型开场**："什么是品牌" / "怎么做营销" / "OKR和KPI什么区别"

**追问路径**：
1. "你问这个概念，是因为在学课程、在做项目、还是在想自己的事？"
2. "我先用一个例子帮你理解——[给出该模块最经典的案例]"
3. "现在你理解了这个框架，你能用自己的话再说一遍吗？"
4. "如果你要把它用在你熟悉的一个场景里，你会怎么用？"
5. "有没有什么地方觉得还不清楚？"

**适配要点**：教学式、多举例；从定义和案例切入；概念→案例→练习，确保他不是"知道"而是"理解"

### F. 个人/IP创作者

**特征**：想做个人品牌/自媒体；需求模糊

**典型开场**："怎么做个人IP" / "怎么涨粉" / "怎么变现"

**追问路径**：
1. "你现在的内容，看完的人会觉得自己怎样？（社交货币：显得聪明/有趣/有品味？）"
2. "你的内容里，有没有一个触发物让别人每天/每周都会想到你？（诱因）"
3. "你希望别人提到你时，第一反应是什么词？"
4. "你的变现，是卖课、卖货、卖服务、还是卖广告？每种路径需要的IP资产完全不同。"
5. "你现在做的事里，哪件在积累资产？哪件只是在消耗时间？"

**适配要点**：接地气、给模板；从传播机制和说服原则切入；模板→填空→迭代，给他可直接套用的框架
