# 诊断恢复模块

> **解决什么问题**：用户想继续之前的诊断，或者查看之前的诊断结论。
> **什么时候用**：用户说"继续""之前的""接着""上次""恢复存档"时。
> **联动文件**：`save.md`（保存存档）/ `report.md`（生成报告）/ `tool-adapter.md`（工具适配）

---

## 触发条件

当用户说以下话术时，触发恢复：

- "继续" / "之前的" / "接着" / "上次" / "恢复存档" / "我的存档" / "看看之前的诊断"

---

## 执行流程

### Step 1：扫描存档目录

**参考 `engine/tool-adapter.md` 检测环境并选择读取方式。**

| 环境 | 操作 |
|------|------|
| 文件可用环境 | 调用 `list_dir`/`list_directory`/`read` 读取 `data/archives/` 目录 |
| 纯对话环境 | 提示用户粘贴存档 JSON |

### Step 2：展示存档列表

> 你有以下存档：
> 1. 张总商业模式诊断 - 2024-01-01 10:00
> 2. 品牌诊断结论 - 2024-01-02 15:30
> 
> 你想恢复哪个？说出编号或名字。

### Step 3：加载存档

用户选择后，读取对应 JSON 文件：

| 环境 | 操作 |
|------|------|
| 文件可用环境 | 调用 `read_file`/`read` 读取选中的 JSON 文件 |
| 纯对话环境 | 解析用户粘贴的 JSON 数据 |

### Step 4：恢复诊断状态

根据存档中的 `phase`、`userProfile` 和 `probingProgress` 字段恢复：

**恢复用户画像**：
- 读取 `userProfile.identity` → 适配语言风格（投资人/顾问视角 vs 创业者视角）
- 读取 `userProfile.industry` → 案例选择偏好
- 读取 `userProfile.stage` → 建议深度适配
- 读取 `userProfile.languageStyle` → 回复风格

**恢复诊断进度**：
- 读取 `realProblem`（摘要，非完整追问记录）→ 已确认真问题时直接从真问题继续
- 读取 `probingProgress` → 真问题未确认时（Phase 1-3 中途保存），用追问进度还原上下文
- 读取 `phase` → 确定从哪个阶段继续：
  - 如果在 Phase 2 之前保存 → 用 `probingProgress.lastTwoRounds` 还原追问上下文，从 `probingProgress.currentLayer` 继续消解
  - 如果在 Phase 4 保存 → 直接展示诊断结论，询问是否需要继续
  - 如果在 Phase 5 保存 → 展示行动方案，询问是否需要执行

> **摘要压缩原则**：恢复时只加载真问题+当前Phase+关键结论+追问进度摘要，不加载完整追问记录。从摘要继续，避免上下文窗口溢出。
>
> **追问进度恢复说明**：若 `probingProgress` 为空且处于 Phase 1-3，说明存档时真问题尚未确认且无追问进度，此时告知用户"上次追问进度未保存，我们从你的初始问题重新开始消解"，不要凭空臆造上下文。

### Step 5：确认继续

展示存档摘要，确认是否继续：

> 存档：张总商业模式诊断
> 时间：2024-01-01 10:00
> 身份：创业者 / 电商 / 规模化期
> 真问题：收入从100万增长到500万
> 诊断阶段：Phase 5（已输出行动方案）
> 
> 要继续吗？还是生成报告？

```
用户：继续我之前的诊断

用户：[粘贴 JSON]
AI：已恢复存档"张总商业模式诊断"。
    真问题：收入从100万增长到500万
    诊断阶段：Phase 5
    要继续吗？
```
