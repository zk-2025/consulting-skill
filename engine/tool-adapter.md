# 工具适配层

> **解决什么问题**：不同 AI 编程环境的文件操作工具名不同。本文件定义统一操作指令，AI 根据当前环境可用的工具自动选择调用方式。
> **主要适配**：WorkBuddy、Claude Code、Codex、Hermes、豆包 + 其他带文件读写的 AI 环境

---

## 环境检测

首次需要文件操作时，检测当前环境是否具备文件读写能力：

| 检测信号 | 环境 | 文件操作能力 |
|----------|------|-------------|
| 有文件写入工具 + 文件读取工具 | **文件可用环境** | 可存档/恢复/生成报告文件 |
| 以上工具都没有 | **纯对话环境** | 降级为对话内输出，提示用户手动保存 |

**检测方法**：在自身可用工具列表中搜索下列工具名（任一即可）。不要猜测，必须确认工具存在才使用。

> ⚠️ **不再区分 WorkBuddy / Claude Code / Codex / Hermes / 豆包 等具体产品**。只要检测到文件读写工具，统一按"文件可用环境"处理；否则按"纯对话环境"降级。按工具能力分两档，执行时不会出现无法判断环境的死锁。

---

## 统一操作接口

### 写文件（write）

整文件写入（存档、报告生成）：

| 环境 | 可用工具名（按优先级） | 参数 | 路径格式 |
|------|----------------------|------|----------|
| 文件可用环境 | `write_to_file` / `write_file` / `write` | `(filePath, content)` | 绝对路径 |
| 纯对话环境 | 无 | 输出到对话，提示用户手动保存 | — |

> **写入工具说明**：部分环境（如 Claude Code、WorkBuddy）的文件写入分两种——`write_to_file`（新建/整文件覆盖）和 `replace_in_file`（编辑现有文件的局部）。存档和报告生成用 `write_to_file`；若需编辑已有存档用 `replace_in_file`。

### 读文件（read）

| 环境 | 可用工具名（按优先级） | 参数 | 路径格式 |
|------|----------------------|------|----------|
| 文件可用环境 | `read_file` / `read` | `(filePath)` | 绝对路径 |
| 纯对话环境 | 无 | 提示用户粘贴内容 | — |

### 列目录（list）

| 环境 | 可用工具名（按优先级） | 参数 | 路径格式 |
|------|----------------------|------|----------|
| 文件可用环境 | `list_dir` / `list_directory` / `read_file`(读目录) | `(target_directory)` | 绝对路径 |
| 纯对话环境 | 无 | 提示用户手动提供文件列表 | — |

### 搜索文件/内容（search）

> 部分环境（如 Claude Code、WorkBuddy）额外提供搜索能力，可用于在存档目录中查找特定存档：

| 操作 | 工具 | 用途 |
|------|------|------|
| 按文件名模式搜索 | `search_file` | 查找 `data/archives/*.json` |
| 按内容搜索 | `search_content` | 在存档内容里找关键词 |

## 降级策略

**核心原则：咨询诊断功能绝不依赖文件操作。文件操作只是增强体验。**

当文件操作不可用时（纯对话环境），优雅降级：

| 功能 | 降级行为 |
|------|----------|
| 存档 | 输出 JSON 到对话，提示"请复制保存为 .json 文件，下次粘贴恢复" |
| 恢复 | 提示"请将之前的存档 JSON 粘贴到对话中" |
| 报告 | 输出 Markdown 到对话，提示"请复制保存为 .md 文件" |

## 各工具特殊适配

| 工具 | 说明 |
|------|------|
| **WorkBuddy** | 支持 Skills 加载，原生文件读写工具，存档目录建议建在 skill 同级 `data/archives/`，报告建在 `data/reports/` |
| **Claude Code** | 自动识别 SKILL.md，内置 `read_file`/`write_to_file` 等文件读写工具 |
| **Codex** | 将 skill 放入 skills 目录即可使用，内置文件读写工具 |
| **Hermes** | 兼容 skill 格式，文件操作通过内置接口，工具名为 `write_file`/`read_file` |
| **豆包** | 支持 Skills 加载，内置文件读写工具 |
| **其他 AI 编程环境** | 只要暴露文件读写工具，按"文件可用环境"处理；工具名不在上述列表时，按工具描述匹配最接近的文件操作能力 |

> **路径格式**：所有环境统一使用绝对路径。
>
> **触发方式差异**：部分平台（如 Hermes、豆包等）支持斜杠命令触发（如 `/dy-skill`）；WorkBuddy/Claude Code 等由 AI 主动读取 `SKILL.md` 及按需加载 engine/modules 文件启动。诊断流程本身完全一致。
