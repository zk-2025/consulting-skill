# 商业咨询skill — 企业商业咨询 AI 系统

> 作者：东野（商业 + AI 博主）｜ 小红书：https://xhslink.com/m/2w27IQZrDzd
>
> 基于系统化商业方法论构建，覆盖战略、运营、组织、营销、品牌、传播、增长全链路诊断。
>
> 适配 WorkBuddy、Claude Code、Codex、Hermes、豆包等支持 Skills 的 AI Agent，不绑定单一工具。

---

## 它是什么

一个可以装进 AI 编程工具的**企业商业咨询系统**。

你不需要打开浏览器搜答案，不需要翻书找框架。在你的 AI 工具里直接对话，它会：

1. **消解你的问题** — 你说的通常不是真问题，先帮你搞清楚真正该问什么
2. **系统诊断** — 用结构化框架拆解问题根源，对照公理检测违反
3. **给出行动方案** — 分层输出 1-3 条可执行的行动，每条有原因+做法+验证信号

## 工作方式（自动选择，你不用选）

你说完第一句话后，AI 会自动判断用哪种方式帮你：

- **有具体问题**（"获客成本太高"）→ 直接追问，帮你找到真问题再解决
- **感觉不对但说不清**（"哪里不太顺"）→ 用几个关键问题帮你定位
- **想全面检查**（"帮我体检一下"）→ 5个维度全景扫描，出诊断报告

**你不需要记这些，正常说话就行。**

## 完整示例：一次诊断长什么样

下面是一个从第一句话到行动方案的完整对话（电商获客案例）：

```
你：我是做电商的，业绩增长遇到瓶颈

AI：📍 第1步/共5步：听你说问题
    你最焦虑的事情是什么？如果只能解决一个，是哪个？

你：获客成本越来越高，投广告越来越亏

AI：📍 第2步/共5步：找到真问题
    你的获客成本大概是多少？一个客户平均给你带来多少收入？

你：获客成本120，客单价80，复购大概2次

AI：📍 第2步/共5步：找到真问题
    也就是说一个客户生命周期价值160，获客要花120？
    那利润只有40，还没算运营成本。这个数字对吗？

你：对，算上运营基本不赚钱

AI：📍 第3步/共5步：确认问题
    所以你的真问题是：LTV/CAC < 1.5，每获一个客只赚一点点，
    广告一涨价就亏。不是"业绩增长遇到瓶颈"，是"单客经济模型不成立"。对吗？

你：对，就是这个

AI：📍 第4步/共5步：诊断根因
    根因：你把增长押在付费获客上，但留存没做起来，LTV撑不起来。
    公理6：先修漏桶，再拉新水——你现在在往漏桶里倒水。

    我诊断的方向是：先提升留存把LTV做上去，再谈拉新。
    这个方向你认可吗？

你：认可

AI：📍 第5步/共5步：给行动方案

    真问题：LTV/CAC < 1.5，单客经济模型不成立

    行动方案：
    1. 先停止扩大付费获客——现在投越多亏越多
       做法：把广告预算砍到当前的30%，省下的钱投到留存
    2. 找出流失节点，提升复购——把LTV从160做到300+
       做法：拉出用户行为路径，找到流失最大的那一步，优先修
    3. 验证信号：LTV/CAC > 3
       如果3个月内这个比值没到3，说明留存方向有问题，回来复诊

    你可以说"保存"存档，或"报告"生成可分享的报告。
    这次诊断对你有帮助吗？
```

> 这只是问诊模式的示例。体检模式会出一份5维度诊断报告，快速扫描会先用5个问题帮你定位方向。

## 特殊能力：戳破"不是问题的问题"

很多用户问的问题，其实不是问题：

| 用户说的 | 真相 | AI怎么处理 |
|----------|------|-----------|
| "这个业务模式行不行？" | 没试过，怕试了失败 | "你试过了吗？试的成本vs不试的代价" |
| "我知道该砍那条线，但是……" | 不愿承认自己判断已清楚 | "你是不是已经知道答案了？什么让你不愿面对？" |
| "团队执行力差" | 目标没对齐，不是人不行 | "你让他们复述过自己的理解吗？点头说知道≠真知道" |

## 安装和使用

### 方式一：克隆到项目目录

```bash
git clone https://github.com/dy-skill/dy-skill.git
```

将克隆的文件放到你的项目根目录，或放到 AI 工具能识别的位置。

### 方式二：各工具配置

本 skill 不绑定单一工具，以下平台均可使用：

| 平台 | 使用方式 |
|------|----------|
| **WorkBuddy** | 将 skill 文件放入 WorkBuddy 的 skills 目录即可加载 |
| **Claude Code** | 将 skill 放入项目目录，自动识别 SKILL.md |
| **Codex** | 将 skill 文件放入 Codex 的 skills 目录即可使用 |
| **Hermes** | 当前文件兼容 Hermes 格式，直接使用 |
| **豆包** | 将 skill 文件放入豆包的 skills 目录即可使用 |
| **其他支持 Skills 的 Agent** | 能读取 SKILL.md 并执行文件读写均可使用 |

### 开始对话

打开你的 AI 工具，直接说：

```
我是做电商的，业绩增长遇到瓶颈
```

或更简单：

```
帮我看看商业模式
```

AI 会自动识别你的问题类型，带你进入诊断流程。

### 常用指令（只需记住这3个）

| 你可以说 | 功能 |
|----------|------|
| "保存" | 保存当前诊断进度 |
| "报告" | 生成诊断报告 |
| "简单说" | 输出精简版 |

> 其余需求正常说话即可，AI 会自动识别（"换个角度""继续上次""直接给方案""帮我看看商业模式"等都不用记）。
