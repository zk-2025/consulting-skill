# 场景：我要做数字化转型

> **典型用户**：企业老板、管理者
> **典型开场**："我要做数字化转型" / "数字化" / "上系统" / "用AI" / "智能化"

---

## 追问引导

1. "你说数字化转型——具体想转什么？哪个业务环节？"
2. "现在那个环节最大的问题是什么？是效率低、成本高、还是信息不透明？"
3. "你算过数字化的ROI吗？投入多少？预期能省多少、赚多少？"
4. "有没有不用数字化也能解决的方案？有时候流程优化就够了。"

---

## 诊断路径

```
用户说："我要做数字化转型"
  → problem-dissolving.md 追问
  → 常见真问题分支：

A. "业务流程效率低" → modules/operations.md（流程效率诊断）→ 先看是不是流程问题不是技术问题
B. "数据驱动决策不够" → modules/operations.md（指标与反馈）→ 先确定需要什么数据
C. "其实是组织问题" → modules/organization.md（目标对齐/执行机制）→ 数字化解决不了人的问题
D. "想用数字化掩盖战略问题" → modules/strategy.md（战略聚焦）→ 工具不能替代方向
E. "ROI算不过来" → modules/business-model.md（成本结构/单位经济模型）→ 不是所有环节都值得数字化
```

---

## 常见误区

1. **数字化是工具不是战略**：先有业务问题，再找工具解决，不是反过来
2. **为了数字化而数字化**：如果ROI算不过来，不上系统才是对的
3. **以为上系统就能提升**：流程本身有问题，上系统只会把低效固化
4. **忽视人的因素**：系统再好，人不配合就是摆设。先解决组织问题再上系统
5. **一步到位**：数字化应该从最痛的环节开始，先跑通一个再扩展

---

## 对标案例

- **亚马逊AWS**：先解决自己的基础设施问题→发现可以开放给外部→成为新业务（内部工具变外部产品）
- **传统零售上系统**：先从库存管理最痛点开始→验证效果→再扩展到CRM→再扩展到全链路（从最痛点开始，逐步扩展）
- **反面**：大量企业花大价钱上ERP但流程没优化→系统成了负担（流程先行，系统后行）
