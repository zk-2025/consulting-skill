# 场景：团队执行力差/目标对不上

> **典型用户**：管理者、总监、创业者
> **典型开场**："团队执行力差" / "目标完不成" / "跨部门协作难" / "OKR落不了地"

---

## 追问引导

1. "你说执行力差——具体差在哪里？能举个最近的例子吗？"
2. "是能力不够、方向不清、还是意愿不足？这三个原因对策完全不同。"
3. "你有没有让团队复述他们对你目标的理解？点头说知道 ≠ 真知道"
4. "你们有OKR吗？每个人知道自己这个季度最重要的1件事是什么吗？"

---

## 诊断路径

```
用户说："团队执行力差"
  → engine/problem-dissolving.md 追问
  → 常见真问题分支：

A. "目标没对齐" → modules/organization.md（OKR：只设1个目标+四象限周会+信心指数）
B. "领导在做下属的事" → modules/organization.md（领导梯队：诊断哪个层级的转型出了问题）
C. "OKR设了但没执行" → modules/organization.md（5个失败因素：优先级/沟通/计划/时间/放弃）
D. "跨部门协作难" → modules/organization.md（OKR自上而下关联+状态指标跨部门同步）
```

---

## 常见误区

1. **"点头说知道"=真知道**：虚假对齐是最常见的执行障碍，需要让团队复述理解
2. **目标太多**：5个目标 = 没有目标 → 只设1个公司级OKR
3. **OKR=KPI**：OKR是目标管理（鼓励挑战），KPI是考核指标（必须达成）→ 混用会让人不敢挑战
4. **没有周一盘点+周五庆祝**：设了OKR就不管了 → 需要承担责任—庆祝成果的节奏
5. **领导者没转型**：还在用前一层级的方式工作 → 领导梯队诊断三个维度（技能/时间/理念）

---

## 对标案例

- **TeaBee创业公司**：第一次设5个OKR全部失败 → 改为只设1个目标+四象限周会+信心指数 → 一季度完成所有关键结果（OKR聚焦原则）
- **事必躬亲的经理**：经理做下属的工作 → 诊断为领导梯队第一阶段转型失败 → 核心问题是工作理念没变：自己干→转为"通过他人"
- **领英CEO杰夫·韦纳**："当你说得不想再说了，人们就开始听进去。" → 目标沟通不是说一次就够，需要渗透到每个角落
