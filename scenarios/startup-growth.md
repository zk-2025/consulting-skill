# 场景：我想获客/增长

> **典型用户**：创业者、产品经理、增长负责人
> **典型开场**："怎么获客" / "我想增长" / "产品怎么推广" / "留存太差"

---

## 追问引导

1. "你的产品，用户用完第一反应是什么？他们会告诉别人吗？"
2. "你现在有用户吗？留存率多少？——如果留存极差，先修漏桶再拉新。"
3. "获客成本是多少？渠道有哪些？哪个渠道ROI最高？"
4. "如果3个月后只有一个指标达标算成功，是什么指标？"

---

## 诊断路径

```
用户说："我想增长"
  → engine/problem-dissolving.md 追问
  → 追问结果可能指向：

A. 传播问题 → modules/spread-and-growth.md（STEPPS检查：有没有社交货币/诱因/公共性？）
B. 漏斗问题 → modules/spread-and-growth.md（AARRR检查：获客/激活/留存/变现哪个环节断？）
C. 说服问题 → modules/influence.md（转化触点是否用了影响力原则？）
D. 品牌问题 → modules/brand-and-marketing.md（有没有CBBE的认知层？）
```

---

## 常见误区

1. **有增长无传播**：用户在增长但没人分享 → 检查STEPPS的公共性/社交货币（参考：Hotmail邮件签名）
2. **只拉人不留人**：获客成本高、留存极差 → 先修漏桶（AARRR的Retention），再拉人（Acquisition）
3. **增长没有节奏**：没有周一盘点+周五庆祝 → 参考OKR四象限周会法（modules/organization.md）
4. **缺增长团队**：增长不是某个部门的事 → 需要跨职能增长团队（产品+工程+营销+数据）

---

## 对标案例

- **Dropbox**：推荐计划（AARRR的Referral）+ 产品内生病毒传播 → 注册量增60%
- **Twitter**：发现"关注30+人后留存显著提升" → 优化激活环节
- **TeaBee**：从5个目标缩减到1个目标+四象限周会 → 一季度后完成所有关键结果
