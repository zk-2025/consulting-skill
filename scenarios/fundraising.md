# 场景：我要融资

> **典型用户**：创业者、企业老板
> **典型开场**："我要融资" / "找投资人" / "需要钱" / "想融一笔"

---

## 追问引导

1. "你融资是为了什么？不是'为了发展'——具体要做什么事？"
2. "你的商业模式跑通了吗？有付费用户吗？留存怎么样？"
3. "如果融不到钱，你的业务能活下去吗？"
4. "你算过需要多少钱吗？这个数字怎么算出来的？"

---

## 诊断路径

```
用户说："我要融资"
  → problem-dissolving.md 追问

D. "想用融资掩盖留存问题" → modules/spread-and-growth.md（先修漏桶）→ 公理6
```

---

## 常见误区

1. **融资是手段不是目的**：融资是为了加速已经验证的模式，不是为了验证模式
2. **先融资再想商业模式**：投资人投的是跑通的模式，不是想做的
3. **融资金额拍脑袋**：需要多少钱=要做什么事×做多久×多少缓冲，不是"越多越好"
4. **以为融资能解决所有问题**：如果留存差/转化率没差异化，钱只会烧得更快

---

## 对标案例

- **美团**：A轮前已验证团购模式有付费用户→融资加速扩张（先验证再融资）
- **瑞幸**：融资前已有完整商业模式设计→融资用于快速铺店（模式先行）
- **反面**：大量O2O项目融资后才发现模式跑不通→钱烧完就死（融资掩盖了模式问题）
