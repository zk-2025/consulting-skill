# 目标清晰化模块

> **解决什么问题**：用户的愿望是模糊的（"我想做个人IP""我想提升业绩"），无法转化为可执行的行动。目标清晰化是把愿望语法审计成可检查的交付物。
> **什么时候用**：用户说"我想做X但不知从何开始""我的目标是成为...""我想变得更.."时。
> **联动文件**：`strategy.md`（战略目标） / `operations.md`（执行目标） / `organization.md`（团队目标）

---

## 核心原则

### 原则1：意义即使用

目标不是被命名的状态，而是一段能在实际生活里驱动行动的语言。脱离使用场景谈目标，就是"语言在度假"。

### 原则2：发动机空转检测

句子里的每个词都必须做工作。不做工作的词就是装饰。

**测试方法**：把这个词去掉，句子是否还成立？如果还成立，这个词就在空转。

常见空转词：**更好、真正的、深入、系统性、全面、有价值、有意义、影响力、好好、认真、长期、持续、打造、建立**。

### 原则3：好目标的五个特征

一个好目标至少满足其中三条：

1. **可指物性** —— 完成时能指着某个东西说"就是这个"
2. **可否证性** —— 存在"没做到"的可能
3. **有完成态** —— 不是永恒进行态
4. **语法健全** —— 谁做、做什么、做到什么程度
5. **嵌在上下文里** —— 与用户当下资源、约束、处境兼容

### 原则4：目标的工作定义

在"帮用户推进行动"这个场景里，目标的工作是让两件事能被确定：

1. **下一步做什么**
2. **什么时候算完**

做不到这两件事的表述，不叫目标，叫愿望语法。

## 诊断流程

### Step 1：接收目标表述

用户可能说：
- "我想做个人IP"
- "我想提升业绩"
- "我的目标是财务自由"
- "我想让团队更有执行力"
- "我想学习商业知识"

### Step 2：发动机空转检测

逐词检查用户的表述，找出空转词：

| 用户说的 | 空转词 | 实际含义 |
|----------|--------|----------|
| "我想做有影响力的内容" | 有影响力 | 具体指什么？10万阅读？1000粉？ |
| "我要认真做好品牌" | 认真、做好 | 什么是"做好"的标准？ |
| "我想长期持续增长" | 长期、持续 | 具体是几个月？什么指标增长？ |
| "我要打造个人IP" | 打造 | 具体做什么？发视频？写文章？ |

### Step 3：五层追问

```
第0层：用户的初始愿望
  ↓ 追问："你说X，你想X具体是指什么？"
第1层：用户对X的具体定义
  ↓ 追问："如果X实现了，你的世界会变成什么样？"
第2层：用户的理想状态
  ↓ 追问："现在阻碍你达到这个状态的最大障碍是什么？"
第3层：用户识别的障碍
  ↓ 追问："这个障碍底下，真正让你卡住的是什么？"
第4层：核心卡点浮现
  ↓ 确认："所以你的核心目标其实是___，对吗？"
```

### Step 4：验证五特征

确认目标后，验证是否满足五个特征：

| 特征 | 检查问题 | 是/否 |
|------|----------|-------|
| 可指物性 | 完成时可以指着什么说"就是这个"？ |  |
| 否证性 | 怎么算"没做到"？ |  |
| 完成态 | 有没有"完成"的那一天？还是永远在路上？ |  |
| 语法健全 | 谁做？做什么？做到什么程度？ |  |
| 上下文嵌合 | 和你当下的资源、时间、能力匹配吗？ |  |

如果不满足，回Step 3继续追问。

### Step 5：输出目标说明书

```markdown
## 你的目标
{一句话精炼}

## 可指物性
完成时能看到：{具体产出}

## 否证性
没做到的表现：{判断标准}

## 完成态
预计完成时间：{具体时间或里程碑}

## 第一步
{具体可执行的第一个动作}
```

## 常见类型的目标审计

### 类型1：个人成长类

用户说："我想提升自己""我想学习"

**审计路径**：
- "提升自己"具体指什么能力？
- 学这个是为了解决什么问题？
- 学到什么程度算"提升了"？
- 这个能力和你的工作/生活有什么关系？

### 类型2：业务增长类

用户说："我想让业绩增长""我想获更多客"

**审计路径**：
- 增长多少？- 什么时间范围内？
- 增长来自哪里？现有客户还是新客户？
- 增长需要什么资源投入？
- 增长的证据是什么？

### 类型3：团队管理类

用户说："我想打造一支有战斗力的团队"

**审计路径**：
- "有战斗力"具体指什么？- 是业绩指标还是团队状态？
- 现在团队差在哪里？
- 你准备投入多少时间/资源？
- 如何衡量"打造完成"？

### 类型4：品牌IP类

用户说："我想做个人IP"

**审计路径**：
- 你想做IP是为了什么？变现？影响力？还是？
- 你计划用什么形式？短视频？文章？直播？
- 你希望别人提到你时说什么？
- 你打算投入多少时间做内容？

### 类型5：财务自由类

用户说："我想财务自由"

**审计路径**：
- 财务自由对你意味着多少钱？
- 这个数字怎么算出来的？
- 你现在有多少？还差多少？
- 通过什么方式达到？

## 常见空转词库

| 空转词 | 意思 | 替换方案 |
|--------|------|----------|
| 提升 | 不具体 | 提升什么？提升多少？ |
| 打造 | 虚 | 具体做什么动作？ |
| 有影响力 | 虚 | 多少粉丝？多少阅读？ |
| 系统性 | 虚 | 具体哪些系统？ |
| 深入 | 虚 | 多深？用什么衡量？ |
| 长期 | 虚 | 具体多久？ |
| 持续 | 虚 | 到什么时候为止？ |
| 规模化 | 虚 | 大到什么程度？ |
| 专业化 | 虚 | 哪些方面专业？ |
| 品牌化 | 虚 | 具体品牌什么要素？ |

## 案例示例

### 案例1：创业者说"我想让公司增长"

| 步骤 | 内容 |
|------|------|
| 原始表述 | "我想让公司增长" |
| 空转词检测 | "增长"——增长多少？增长什么？ |
| 追问 | "增长是指收入增长还是用户增长？" |
| 用户回答 | "收入增长" |
| 继续追问 | "收入增长多少？一年后希望达到什么规模？" |
| 用户回答 | "希望从100万增长到500万" |
| 追问 | "增长到500万打算怎么做到？" |
| 用户回答 | "拓展新客户" |
| 继续追问 | "新客户从哪来？什么渠道？获客成本多少？" |
| 目标确认 | "你的核心目标是：一年时间，让公司收入从100万增长到500万，通过拓展新客户实现" |
| 五特征验证 | ✅ 可指物（500万收入）✅ 否证性（没达到就是没做到）✅ 完成态（一年后）✅ 语法健全✅ 上下文嵌合 |

### 案例2：管理者说"我想打造有执行力的团队"

| 步骤 | 内容 |
|------|------|
| 原始表述 | "我想打造有执行力的团队" |
| 空转词检测 | "有执行力"——具体指什么？ |
| 追问 | "你说执行力，具体表现是什么？" |
| 用户回答 | "布置的任务能按时完成" |
| 继续追问 | "所有任务还是关键任务？按时是多久？" |
| 用户回答 | "关键任务，按时是指在约定时间内" |
| 追问 | "现在团队有多少人？有几个人做不到按时完成？" |
| 用户回答 | "团队10人，有3人经常延期" |
| 目标确认 | "你的核心目标是：让团队10人中的关键任务按时完成率从70%提升到95%，3个经常延期的人不再延期" |

## 行动方案模板

```markdown
## 目标清晰化行动方案

### 第1步：空转词清理
- 原始表述：{用户说的}
- 空转词：{哪些词在做空转}
- 清理后：{去掉空转词后的表述}

### 第2步：五特征验证
- 可指物性：{完成时能指着什么}
- 否证性：{怎么算没做到}
- 完成态：{什么时候算完}
- 语法健全：{谁做+做什么+做到什么程度}
- 上下文嵌合：{和当下资源匹配吗}

### 第3步：第一步行动
- 具体动作：{现在就能做的第一个动作}
- 验证信号：{什么现象说明方向对了}
```

## 模块联动声明

- **诊断完成后通常需要检查**：`strategy.md`（目标是否服务于战略方向）、`organization.md`（目标如何拆解为 OKR 和执行计划）
- **如果用户同时追多个目标**：用“如果只能保一个”的追问锁定优先级
- **如果目标确认后需要落地**：路由到 `operations.md` 建立执行机制
- **跨模块冲突**：当个人目标与组织战略不一致时，先对齐再行动——“如果所有事情都同等重要，就意味着它们也同等不重要”
