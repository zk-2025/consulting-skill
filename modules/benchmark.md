# 对标分析模块

> **解决什么问题**：用户想模仿优秀案例，但不知道该模仿谁、模仿什么、怎么模仿。对标分析是帮你找到值得模仿的对标，排除一切噪音。
> **什么时候用**：用户说"我想模仿谁""我该学谁""有没有成功的案例可以抄"时。
> **联动文件**：`strategy.md`（战略对标） / `spread-and-growth.md`（增长对标） / `brand-and-marketing.md`（品牌对标）

---

## 核心原则

### 原则1：对标的本质是找差距，不是抄作业

对标不是找一个"成功的榜样"然后全套复制，而是找到**你和他的差距在哪里**，然后针对性地学习能学的那部分。

### 原则2：五重过滤找对标

找一个值得对标的人/公司，需要过五层过滤：

| 过滤层 | 问题 | 目的 |
|--------|------|------|
| **第一层：业务相似** | 他做的和你做的是同一件事吗？ | 业务不相似，对标没意义 |
| **第二层：阶段相似** | 他在什么阶段成功的？你在什么阶段？ | 阶段不同，方法不同 |
| **第三层：资源相似** | 他有什么资源？你有什么资源？ | 资源差距太大，复制不了 |
| **第四层：可学习** | 他的成功是因为能力还是运气？ | 运气学不了，能力可以学 |
| **第五层：可落地** | 你准备投入多少？愿意放弃什么？ | 不愿意付出，对标是空谈 |

### 原则3：对标的是系统，不是表面

抄产品、抄文案、抄功能都是抄表面。真正值得对标的是：
- 他是怎么发现这个机会的？
- 他是怎么验证这个想法的？
- 他是怎么组建团队的？
- 他是怎么获取第一批用户的？
- 他是怎么解决从0到1的瓶颈的？

## 诊断流程

### Step 1：明确对标目的

用户需要回答：你想通过对标解决什么问题？

| 目的 | 推荐的标的类型 |
|------|---------------|
| 战略方向不确定 | 行业头部公司 |
| 增长遇到瓶颈 | 相似体量的成功公司 |
| 产品没差异化 | 竞品或跨行业创新者 |
| 品牌不知道怎么打 | 相似受众的成功品牌 |
| 团队管理有问题 | 管理优秀的公司 |

### Step 2：五重过滤

#### 第一层：业务相似过滤

问题：他和你是做同一件事吗？

**判断标准**：
- 目标客户相同
- 解决的问题相同
- 提供的价值相同

如果业务不相似，直接排除。

#### 第二层：阶段相似过滤

问题：他在什么阶段成功的？你在什么阶段？

**阶段分类**：
- 0-1（验证期）：想法还没验证
- 1-10（增长期）：模式已验证，需要规模化
- 10-100（成熟期）：已经规模化，需要精细化

如果阶段差太大（比如他在成熟期而你在0-1），他的方法对你不适用。

#### 第三层：资源相似过滤

问题：他有什么资源？你有什么资源？

需要对比：
- 资金实力
- 团队规模
- 人脉关系
- 时间投入

如果资源差距太大（比如他融资了1个亿而你还在用自有资金），他的做法你学不了。

#### 第四层：可学习过滤

问题：他的成功是因为能力还是运气？

**判断方法**：
- 他成功之前做了什么积累？
- 他有什么独特的技能或资源？
- 他的成功是偶然还是必然？

如果纯粹是运气（Timing好、遇到贵人），那不值得对标。

#### 第五层：可落地过滤

问题：你愿意为他付出的代价付出吗？

需要回答：
- 你愿意投入多少时间？
- 你愿意投入多少钱？
- 你愿意放弃什么？

如果不愿意付出，对标是空谈。

### Step 3：差距分析

找到对标后，分析三个差距：

| 差距类型 | 问题 |
|----------|------|
| **能力差距** | 他会什么我不会？ |
| **资源差距** | 他有什么我没有？ |
| **认知差距** | 他知道什么我不知道？ |

### Step 4：制定学习计划

从差距中提取可学习的点，制定学习计划：

```markdown
## 对标分析报告

### 标的
{公司/人名}

### 为什么选他
1. 业务相似度：{匹配/不匹配}
2. 阶段相似度：{匹配/不匹配}
3. 资源相似度：{匹配/不匹配}

### 核心差距
- 能力差距：{具体}
- 资源差距：{具体}
- 认知差距：{具体}

### 可学习的点（按优先级）
1. {第一优先级学习点} - 怎么学
2. {第二优先级学习点} - 怎么学
3. {第三优先级学习点} - 怎么学

### 第一步行动
{具体可执行的第一个动作}
```

## 常见对标场景

### 场景1：我想做某个产品，有人做得好的吗？

**过滤流程**：
1. 先问：你的产品和他的一样吗？目标用户一样吗？
2. 再问：他在什么阶段做的？你在什么阶段？
3. 再问：他投入了多少？你有多少？
4. 再问：他成功是因为什么？能力还是运气？
5. 再问：你愿意付出和他一样的代价吗？

### 场景2：我想做某个领域的市场，谁能抄？

**过滤流程**：
1. 先问：你想做这个领域的什么位置？老大还是老三？
2. 再问：谁现在做得好？他做对了什么？
3. 再问：他做对的事你能不能做？需要什么条件？
4. 再问：你准备用多长时间追赶？

### 场景3：我的增长遇到瓶颈了，谁能帮我突破？

**过滤流程**：
1. 先问：你的瓶颈是什么？获客？转化？留存？
2. 再问：谁也遇到过这个瓶颈？他怎么解决的？
3. 再问：他解决问题的方法你需要什么条件？
4. 再问：你能接受他解决问题的方法吗？

## 案例示例

### 案例：创业者想对标瑞幸咖啡

| 步骤 | 内容 |
|------|------|
| 用户问题 | "我想做咖啡品牌，瑞幸可以做我对标吗？" |
| 第一层过滤 | 业务相似？都是做咖啡的→✅ 通过 |
| 第二层过滤 | 阶段相似？瑞幸已经上市了，用户还在0-1验证期→❌ 不通过 |
| 结论 | 不适合对标瑞幸，阶段差太大 |
| 建议 | 找一个也是刚起步但已经验证了模式的咖啡品牌对标 |

### 案例：想做内容创业，有人可以抄吗？

| 步骤 | 内容 |
|------|------|
| 用户问题 | "我想做知识IP，有人可以抄吗？" |
| 第一层过滤 | 业务相似？都是做知识内容→✅ 通过 |
| 第二层过滤 | 阶段相似？需要看用户在什么阶段 |
| 用户回答 | 我刚起步，还没开始做 |
| 第三层过滤 | 资源相似？需要问投入多少时间/钱 |
| 用户回答 | 我兼职做，每天2小时 |
| 第四层过滤 | 能力vs运气？全职IP和兼职IP的成功路径不同 |
| 结论 | 不建议对标头部IP，他们的方法兼职做不来 |
| 建议 | 找同阶段（兼职起步）的IP做对标，学习怎么从0开始 |

## 对标搜索方法

### 1. 行业搜索
- 行业协会报告
- 行业媒体榜单
- 投资机构投资清单

### 2. 竞品搜索
- 你的竞品是谁？
- 竞品的竞品是谁？
- 跨行业类似模式

### 3. 人才搜索
- 你的偶像是谁？
- 你行业里佩服的人是谁？
- 他们之前在哪里工作？做什么？

### 4. 案例搜索
- 创业媒体采访
- 商业案例分析
- 公开演讲/分享

## 模块联动声明

- **诊断完成后通常需要检查**：`strategy.md`（对标结论如何转化为自身战略）、`business-model.md`（对标对象的商业模式是否可迁移）
- **如果涉及行业/赛道选择**：补充查看 `case-library.md` 按行业和阶段检索案例
- **如果用户想模仿某个具体品牌**：用五重过滤法验证是否真的适合模仿
- **跨模块冲突**：对标分析是手段不是目的——最终要回到自身的问题和资源约束
